# دليل كتابة المقالات العلمية

هذا الدليل يشرح لك كيف تكتب مقالات الرياضيات والهندسة من داخل أوبسيديان (Obsidian) لتحصل على التصميم الاحترافي الجديد تلقائياً بمجرد النشر في Quartz.

---

## ١. مكان حفظ المقال (شرط أساسي)
لكي يتم تطبيق التصميم الجديد واللغة العربية بشكل صحيح وتلقائي، **يجب** أن تحفظ المقال أو تنقله إلى أحد المجلدات التالية داخل مجلد `content`:
- `math` أو `رياضيات`
- `engineering` أو `هندسة`

بمجرد وضع الملف في أحد هذه المجلدات، سيقوم Quartz بتفعيل التصميم الجديد بالكامل.

---

## ٢. العناوين والنصوص
اكتب النصوص والعناوين بشكل طبيعي جداً باستخدام الـ Markdown المعتاد.
- `# عنوان رئيسي` (سيتم وضع الخط الأصفر أسفله تلقائياً ويأخذ نوع الخط المخصص).
- `## عنوان فرعي`
- لكتابة خط فاصل مزخرف (الخط الذي يتوسطه المعين ◆)، اكتب فقط ثلاث شرطات `---`.

---

## ٣. المعادلات الرياضية (LaTeX)
اكتب المعادلات بصيغة أوبسيديان الافتراضية، وسيقوم التصميم الجديد بتحويلها للشكل الجميل تلقائياً (خلفية بيضاء، إطار، محاذاة صحيحة للغة الإنجليزية).

**معادلة معزولة (Block):**
اكتب `$$` ثم المعادلة ثم `$$`:
$$
\lim_{x \to 0} \frac{\sin x}{x} = 1
$$

**معادلة مضمنة (Inline):**
استخدم علامة الدولار المفردة `$x^2 + 1$` وسط النص.

---

## ٤. استخدام الصناديق الملونة (Callouts)
قمت بربط نظام الـ Callouts الخاص بأوبسيديان/Quartz بالصناديق الملونة في التصميم الجديد. يمكنك كتابتها هكذا:

**الصندوق الأبيض (ملاحظة/فكرة عادية):**
> [!note] عنوان الملاحظة
> هذا النص سيظهر داخل صندوق أبيض بظل خفيف.

**الصندوق الأزرق (معلومة/إضافة):**
> [!info] معلومة مهمة
> يظهر هذا الصندوق بخلفية زرقاء هادئة.

**الصندوق الأحمر (تحذير/خطأ/تنبيه):**
> [!warning] انتبه
> أو استخدم `[!danger]` أو `[!bug]`. يظهر بخلفية حمراء.

**الصندوق الأخضر (نجاح/مثال إيجابي):**
> [!success] صحيح
> أو استخدم `[!tip]`. يظهر باللون الأخضر/الزيتي.

**الصندوق الأصفر (سؤال/مثال/مفهوم):**
> [!example] مثال رقم ١
> أو استخدم `[!question]`. يظهر باللون الأصفر المميز للتصميم.

## ٥. إضافة عناصر متقدمة بالـ HTML

> [!warning] قاعدة ذهبية: LaTeX لا يعمل داخل HTML
> محرك Quartz يعالج الـ LaTeX (`$...$` و `$$...$$`) فقط في سياق الماركدوان العادي.
> إذا وضعت LaTeX داخل `<div>` أو أي عنصر HTML — **لن يتم تحويله** وسيظهر كنص خام!

### الطريقة الصحيحة لكتابة تعريف رياضي أو معادلة مميزة:

> [!tip] تمكين التوافق مع الجوال (Responsive Math)
> المعادلة المكتوبة داخل الـ Callouts بصيغة `$$` مدعومة الآن بالكامل لتكون متجاوبة مع شاشات الجوال. إذا كانت المعادلة طويلة جداً (مثل صيغة نيوتن)، سيتم تصغيرها قليلاً مع إمكانية التمرير الأفقي الناعم دون تشويه تصميم الصفحة أو ظهور أشرطة تمرير مزعجة.

**✅ صحيح — استخدم Callout (وهو الشكل المعتمد للأمثلة والتعريفات الطويلة):**
```markdown
> [!example] صيغة نيوتن — القول الأصلي
> $$ \frac{ds}{dt} = \lim_{dt \to 0} \frac{s(t + dt) - s(t)}{dt} $$
```

**❌ خاطئ — لا تضع LaTeX داخل HTML:**
```html
<!-- هذا لن يعمل في Quartz! ولن يظهر بشكل جيد على الجوال -->
<div class="mblock">
  $$ \lim_{x \to c} f(x) = L $$
</div>
```

### عناصر HTML المسموح بها (بدون LaTeX بداخلها):
يمكنك استخدام HTML لعناصر الهيكلة البحتة مثل الشبكات والجداول:

**الشبكة المزدوجة للمقارنات (بدون LaTeX):**
```html
<div class="gg">
  <div class="pc re">
    <div class="pn">الخصم — يختار ε</div>
    <p>يختار الخصم هامش خطأ ليثبت فشلك.</p>
  </div>
  <div class="pc te">
    <div class="pn">أنت — تجد δ</div>
    <p>بعد اختيار الخصم، تجد أنت النطاق المناسب.</p>
  </div>
</div>
```

**اقتباس أدبي (Epigraph):**
```html
<div class="epq">
  <p>نص الاقتباس هنا.</p>
  <div class="att">— اسم المؤلف</div>
</div>
```

## ٦. الرسومات التفاعلية (Canvas)

الموقع يتضمن سكربت `math-canvas.js` يرسم رسومات بيانية تفاعلية على عناصر `<canvas>`. لإضافة رسم تفاعلي جديد:

### الهيكل الأساسي:
```html
<div class="dw">
  <div class="dtit">وصف الرسم البياني</div>
  <canvas id="myCanvasId" width="760" height="350" class="dc"></canvas>
  <div class="cr"><span class="cl">اسم المتغير</span><input type="range" id="mySliderId" min="0" max="5" step="0.1" value="2"><span class="cn" id="myValueId">2.00</span></div>
  <div class="ir">
    <div class="ic"><div class="icl">عنوان القيمة ١</div><div class="icv" style="color:var(--amber)" id="myVal1">0.0000</div></div>
    <div class="ic"><div class="icl">عنوان القيمة ٢</div><div class="icv" style="color:var(--teal)" id="myVal2">0.0000</div></div>
  </div>
</div>
```

### الأمثلة التفاعلية الموجودة حالياً:
- **مثال ١** (f(x) = x²): Canvas ID = `ex1C`، Slider ID = `ex1S`
- **مثال ٢** (ثقب في الدالة): Canvas ID = `ex2C`، Slider ID = `ex2S`
- **مثال ٣** (sin(x)/x): Canvas ID = `ex3C`، Slider ID = `ex3S`
- **مثال ٤** (نهاية غير موجودة): Canvas ID = `ex4C`، Slider ID = `ex4S`
- **الجوار المفرغ**: Canvas ID = `nlC`

لإضافة مثال تفاعلي جديد، أضف كود الرسم في ملف `quartz/static/math-canvas.js` داخل دالة `initMathCanvas`.

## ⚠️ قواعد مهمة لكتابة HTML داخل Markdown

> [!warning] لا تترك أسطراً فارغة داخل أكواد HTML
> محرك Quartz يتعامل مع الأسطر الفارغة داخل HTML على أنها بداية فقرة جديدة ويغلفها في `<p>` مما يكسر التنسيق.

**❌ خاطئ:**
```html
<div class="dw">

  <div class="dtit">عنوان</div>

  <canvas id="myC" width="760" height="350" class="dc"></canvas>

</div>
```

**✅ صحيح:**
```html
<div class="dw">
  <div class="dtit">عنوان</div>
  <canvas id="myC" width="760" height="350" class="dc"></canvas>
</div>
```

> [!note] ملاحظة عن الـ Frontmatter
> تأكد من وجود **frontmatter واحد فقط** في بداية الملف (بين `---` و `---`). تكرار الـ frontmatter سيظهر كنص عادي في الصفحة.
